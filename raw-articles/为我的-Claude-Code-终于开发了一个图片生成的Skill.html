为 ClaudeCode 集成了 Image Generate Skill

我今天花了下午的时间，终于给我的ClaudeCode 做了一个 Image Generate Skill 工具

在写公众号文章或者博客文章的时候，有一个环节让我非常沮丧，也比较恼火，就是给文章配封面图。之前的工作流程是：

  1. 文章写好后，我会把文章内容丢给 AI，让它帮我基于文章内容帮我写出一个封面图的 prompt。
  2. AI 会把 prompt 给到我，然后我再把这个 prompt 用于生成封面图。
  3. 生成封面图时，通常会打开即梦，或者有时候会用豆包平台，让它们帮我生成封面图。

在这个过程中，需要设置很多参数，有时候我也记不住。而且 AI 其实也不太能记住，比如豆包平台的模型 prompt 输入参数是怎样的。反正来来回回要做很多调整，这里面其实要花不少时间。

我今天终于做了一个工具，给 ClaudeCode 做了一个技能（Skill）——图片生成技能（Image Generator Skill）。

因为我之前在使用 Claude 的时候，我发现 Claude 它就有这个文生图的模型技能，但是我发现 ClaudeCode 它竟然没有。主要原因还是因为 ClaudeCode 它没有这个文生图模型，对，Claude（或者说 Anthropic）它没有文生图模型。那没有的话，我就自己造了一个。

现在这个技能可以去集成各种平台。我目前暂时只集成了一个平台：

  • 一个是火山引擎这个平台。

后期的话，如果说我要让它去集成一些第三方平台，都是非常容易的，因为它整个框架就是一个非常容易去接入一个平台的框架。

现在，如果我要给某篇文章生成一个封面图的话，很多时候就是一句话。我直接在 ClaudeCode 里把这个文件路径给它，然后我就说“给我生成一个文章的封面图”。它很快就能理解这个东西怎么去做，然后去调用相应的技能，并且把这个图片还帮我自动下载到本地。对，然后很快就解决了。

而且，因为我是通过API调用的，生成的图片没有水印，也省去了我去除水印的过程，所以说非常方便，又帮我解决了一个问题。

后续，我可以把这个Skill分享出来。它的集成方式其实非常简单。我会在之后更新一篇文章来说明，这个Skill的文件结构是怎样的，以及它的实现逻辑。Skill
