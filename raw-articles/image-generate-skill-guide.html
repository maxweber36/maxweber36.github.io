image-generate Skill 讲解指南

一句话概括

这个 Skill 让 Claude 拥有了”画画”能力——你用文字描述想要什么图，它调用云端 AI 模型生成图片并保存到你本地。

文件结构

整个 Skill 只有 3 个文件，结构非常简洁。

各文件的作用

1. SKILL.md — 总指挥

这是 Skill 的入口文件，分为两部分：

Frontmatter（文件头）

  • name: Skill 的标识名，在 Claude 中用 /image-generate 触发
  • description: 告诉 Claude 什么时候该用这个 Skill。你提到”生成图片””文生图””帮我画””文章封面”等关键词时，Claude 就会自动激活它

正文 — 五步工作流

步骤 做什么 关键点
Step 1 确认需求 从你的输入中提取 prompt、用途、尺寸偏好
Step 2 选择平台 根据你的需求自动选最合适的 AI 平台
Step 3 调用 API 读取平台文档，构造请求，获取图片
Step 4 下载保存 把 URL 或 base64 图片存到本地文件
Step 5 返回结果 告诉你保存路径、尺寸、用的什么模型

正文还包含：

  • 平台选择逻辑：一段伪代码，教 Claude 如何根据场景选平台
  • 代码模板：call_platform_api() 和 download_and_save() 两个 Python 函数骨架
  • Prompt 优化建议：自动为你的描述补充质量关键词（如 “illustration style, clean background”）
  • 集成新平台指南：只需加一个 reference 文件 + 改两行表格

2. references/{platform}.md — 各平台 API 手册

每个 reference 文件是一个平台的完整 API 参考文档，包含：

  • 认证方式（通常为 Bearer Token，通过环境变量获取）
  • API 端点和请求方法
  • 所有请求参数的详细说明（model、prompt、size、watermark 等）
  • 推荐尺寸对照表（16:9 封面、1:1 配图等）
  • 可直接使用的 Python 调用代码

部分平台文件可能是模板状态（骨架已搭好，待补充具体 API 信息后即可启用）。

运行流程（用大白话说）

设计亮点

  1. 平台解耦：每个 AI 平台独立一个 reference 文件，互不干扰。新增平台只需加文件 + 改两行表格
  2. 按需加载：Claude 只在你选择某平台时才读取对应的 reference 文件，不会把所有 API 文档塞满上下文
  3. 无硬编码密钥：API Key 全部通过环境变量传递，安全且灵活
  4. 智能推荐：内置平台选择逻辑，根据用途自动推荐最合适的模型和尺寸

如何使用

  1. 确保已设置对应平台的环境变量（如 export YOUR_PLATFORM_API_KEY=你的key）
  2. 在 Claude Code 中直接说”帮我生成一张…”
  3. 或使用 /image-generate 命令显式触发

如何扩展新平台

  1. 在 references/ 下新建 {平台名}.md，照着已有 reference 文件的格式写 API 文档
  2. 在 SKILL.md 的”支持平台”表格中加一行
  3. 在”认证配置”表格中加环境变量名
  4. 完成
